8.1.1 理论基础

回溯法

① 回溯法：是一种搜索的方式。回溯是递归的副产品，只要有递归就会有回溯。回溯函数也就是递归函数，指的都是一个函数。

② 性能分析：虽然回溯法很难，很不好理解，但是回溯法并不是什么高效的算法。因为回溯的本质是穷举。

③ 如何理解：既然并不高效，为什么还要用它呢？因为没得选，一些问题能暴力搜出来就不错了，撑死了再剪枝一下。

解决的问题

  1. 组合问题：N 个数里面按一定规则找出 k 个数的集合

  2. 切割问题：一个字符串按一定规则有几种切割方式

  3. 子集问题：一个 N 个数的集合里有多少符合条件的子集

  4. 排列问题：N 个数按一定规则全排列，有几种排列方式

  5. 棋盘问题：N 皇后，解数独等等

这些问题都可以抽象为树形结构，因为都是在集合中递归查找子集。集合大小构成了树的宽度，递归深度就构成了树的深度。

回溯法模板

void backtracking(参数) {
    if (终止条件) {
        // 存放结果
        return;
    }
    for (遍历本层集合元素) {
        // 1. 处理
        backtracking(参数); // 2. 递归
        // 3. 回溯
    }
}

class Solution {
    private List<List<Integer>> paths = new ArrayList<>();
    private List<Integer> path = new ArrayList<>();


    public List<List<Integer>> combine(int n, int k) {
        backtracking(n, k, 1);
        return paths;
    }


    private void backtracking(int n, int k, int start) {
        if (path.size() == k) {
            paths.add(new ArrayList<>(path));   // 存放结果
            return;
        }                                       // 未剪枝: i <= k
        for (int i = start; i <= n - (k - path.size()) + 1; i++) {  
            path.add(i);                        // 1. 处理
            backtracking(n, k, i + 1);          // 2. 递归
            path.removeLast();                  // 3. 回溯
        }
    }
}

private void backtracking(int n, int k, int sum, int start) {
    if (sum > n) return; 
    if (path.size() == k) {
        if (sum == n) {
            paths.add(new ArrayList<>(path));
        }
        return;
    }       
    for (int i = start; i <= 9 - (k - path.size()) + 1; i++) {
        sum += i;
        path.add(i);
        backtracking(n, k, sum, i + 1);
        sum -= i;
        path.removeLast();
    }
}

    private StringBuilder path = new StringBuilder();
    private String[] letters = {

    };          // 1. error: List<String>

    }

class Solution {
    private List<List<Integer>> paths = new ArrayList<>();
    private List<Integer> path = new ArrayList<>();


    public List<List<Integer>> combinationSum(int[] nums, int target) {
        Arrays.sort(nums);

    if (sum == target) {

private void backtracking(String s, int start) {
    if (start >= s.length()) {
        paths.add(new ArrayList<>(path));
        return;
    }
    for (int i = start; i < s.length(); i++) {
        if (isPalindrome(s, start, i)) {
            String rest = s.substring(start, i + 1);    // error: subString

private boolean isPalindrome(String s, int left, int right) {
    while (left < right) {
        if (s.charAt(left) != s.charAt(right)) {
            return false;
        }
        left++;
        right--;
    }
    return true;
}

8.2.2 复原 IP 地址

力扣题目链接

有效 IP 地址 正好由四个整数（每个整数位于 0 到 255 之间组成，且不能含有前导 0），整数之间用 '.' 分隔。

  • 例如："0.1.2.201" 是 有效 IP 地址，但是 "0.011.255.245"、"192.168.1.312" 是 无效 IP 地址。

给定一个只包含数字的字符串 s ，用以表示一个 IP 地址，返回所有可能的有效 IP 地址，可以通过在 s 中插入 '.' 来形成。

你 不能 重新排序或删除 s 中的任何数字。你可以按 任何 顺序返回答案。

xxxxxxxxxx
输入：s = "101023"
输出：["1.0.10.23","1.0.102.3","10.1.0.23","10.10.2.3","101.0.2.3"]

  • 思路

判断段是否是有效段位：① 以 0 开头不合法；② 大于 255 不合法。

  • 实现

xxxxxxxxxx
class Solution {
    private List<String> result = new ArrayList<>();


    public List<String> restoreIpAddresses(String s) {
        if(s.length() < 4 || s.length() > 12) return result;
        backtracking(s, 0, 0);
        return result;
    }


    private void backtracking(String s, int start, int count) {
        if(count == 3){
            if(isValid(s, start, s.length() - 1)) result.add(s);
            return;
        }
        for(int i = start; i < s.length(); i++){
            if(isValid(s, start, i)){
                // 1. error: s.substring(start, i + 1)
                s = s.substring(0, i + 1) + "." + s.substring(i + 1);  
                count++;
                // 2. error: s.substring(s, i + 1, count);
                backtracking(s, i + 2, count);
                s = s.substring(0, i + 1) + s.substring(i + 2);
                count--;
            } else{
                break;  // 3. don't forget break
            }
        }
    }


    private boolean isValid(String s, int begin, int end) {
        if(begin > end) return false;
        if(s.charAt(begin) == '0' && begin < end) return false;
        int num = 0;
        for(int i = begin; i <= end; i++){
            num = num * 10 + (s.charAt(i) - '0'); // 4. don't forget ()
        }
        return num <= 255;
    }
}

    public List<List<String>> solveNQueens(int n) {
        char[][] board = new char[n][n];
        for (int i = 0; i < n; i++) {

        }

    public boolean backTracking(List<List<String>> tickets, boolean[] used) {
        if (path.size() == tickets.size() + 1) {
            result = new ArrayList(path);   // 1. error: result.add
            return true;
        }
        for (int i = 0; i < tickets.size(); i++) {
            if (used[i] == false) {         // 2. error: get(0)
                if(tickets.get(i).get(0).equals(path.getLast())){
                    used[i] = true;
                    path.add(tickets.get(i).get(1));
                    if (backTracking(tickets, used)) return true;
                    used[i] = false;
                    path.removeLast(); 
                }
            }
        }
        return false;                       // 3. error: true
    }
}

8.6.3 总结

组合问题

时间复杂度：O(2^n)，组合问题就是特殊的子集问题，所以最坏情况不会超过子集问题的时间复杂度。空间复杂度：O(n)。

子集问题

时间复杂度：O(2^n)，因为每一个元素的状态无外乎取与不取，所以时间复杂度为 O(2^n)。空间复杂度：O(n)。

排列问题

时间复杂度：O(n!)，第一层 n 个分支，第二层 n-1 个分支， ... ，一直到叶子节点一共就是 n! 。空间复杂度：O(n)。

N皇后问题

时间复杂度：O(n!) ，直觉上是 O(n^n)，但皇后之间不能同列所以有剪枝，最差情况是 O(n!)。空间复杂度：O(n)。

解数独问题

时间复杂度：O(9^m)，其中 m 是 '.' 的数目。空间复杂度：O(n^2)，递归深度为 n^2。

9.1 简单题目

9.1.1 理论基础

贪心

贪心的本质是选择每一阶段的局部最优，从而达到全局最优。例1：有一堆钞票可以拿走十张，如果想达到最大的金额要怎么拿？

指定每次拿最大的，最终结果就是拿走最大数额的钱。每次拿最大的就是局部最优，最后拿走最大数额的钱就是推出全局最优。

例2：如果是有一堆盒子，背包体积为 n，如何把背包尽可能装满？如果还每次选最大的盒子就不行了，这时候就需要动态规划。

贪心的使用

贪心算法没有固定的套路。那如何验证能否用贪心呢？最好用的策略就是举反例，如果想不到反例，就试一试贪心吧。

class Solution {
    public int largestSumAfterKNegations(int[] A, int K) {
        List<Integer> B = new ArrayList<>();
        for (int x : A) B.add(x);
        Collections.sort(B, (a, b) -> Integer.compare(Math.abs(b), Math.abs(a)));


        for (int i = 0; i < B.size(); i++) {
            if (B.get(i) < 0 && K > 0) {
                B.set(i, -B.get(i));
                K--;
            }
        }

            B.set(B.size() - 1, -B.getLast());
        }
        int sum = 0;
        for (int x : B) sum += x;
        return sum;
    }
}
