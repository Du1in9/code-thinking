数组是非常基础的数据结构，在面试中，考察数组的题目一般在思维上都不难，主要是考察对代码的掌控能力。

一维数组

数组是存放在连续内存空间上的相同类型数据的集合。

数组可以方便的通过下标索引的方式获取到下标下对应的数据。需要注意的是：

  • 数组下标都是从 0 开始的

  • 数组内存空间的地址是连续的

所以我们在增删元素的时候，就难免要移动其他元素的地址。数组的元素是不能删的，只能覆盖。

示例 2:

xxxxxxxxxx
输入: nums = [-1,0,3,5,9,12], target = 2     
输出: -1        
解释: 2 不存在 nums 中因此返回 -1        

  • 思路

这道题目的前提是数组为有序数组，同时题目还强调数组中无重复元素。

因为一旦有重复元素，使用二分查找法返回的元素下标可能不是唯一的，这些都是使用二分法的前提条件。

class Solution {
    public int search(int[] nums, int target) {
        if (target < nums[0] || target > nums[nums.length - 1]) {
            return -1;
        }
        int left = 0, right = nums.length - 1;
        while (left <= right) {         // 1. <=, not <
            int mid = (left + right) / 2;
            if (nums[mid] == target)
                return mid;
            else if (nums[mid] < target)
                left = mid + 1;         // 2. mid + 1, not mid
            else if (nums[mid] > target)
                right = mid - 1;        // 3. mid - 1, not mid
        }
        return -1;
    }
}

1.5 长度最小的子数组

  • 题目

力扣题目链接

给定一个含有 n 个正整数的数组和一个正整数 s ，找出该数组中满足其和 ≥ s 的长度最小的 连续 子数组，并返回其长度。如果不存在符合条件的子数组，返回 0。

示例：

  • 输入：s = 7, nums = [2,3,1,2,4,3]

  • 输出：2

  • 解释：子数组 [4,3] 是该条件下的长度最小的子数组。

  • 思路

接下来就开始介绍数组操作中另一个重要的方法：滑动窗口。

所谓滑动窗口，就是不断的调节子序列的起始位置和终止位置，从而得出我们要想的结果。

  • 实现

xxxxxxxxxx
class Solution {
    public int minSubArrayLen(int target, int[] nums) {
        int left = 0, sum = 0;
        int result = Integer.MAX_VALUE;
        for (int right = 0; right < nums.length; right++){
            sum += nums[right];
            while (sum >= target){
                result = Math.min(result, right - left + 1);
                sum -= nums[left++];
            }
        }
        return result == Integer.MAX_VALUE ? 0 : result;    // 0, not Integer.MAX_VALUE
    }
}

力扣题目链接

给定一个正整数 n，生成一个包含 1 到 n^2 所有元素，且元素按顺时针顺序螺旋排列的正方形矩阵。

示例: 输入: 3 输出: [ [ 1, 2, 3 ], [ 8, 9, 4 ], [ 7, 6, 5 ] ]

  • 思路

这道题目可以说在面试中出现频率较高的题目。

本题并不涉及到什么算法，就是模拟过程，但却十分考察对代码的掌控能力。

  • 实现

xxxxxxxxxx
class Solution {
    public int[][] generateMatrix(int n) {
        int [][] nums = new int[n][n];
        int startX = 0, startY = 0;
        int loop = 1, count = 1, offset = 1;
        int i, j;
        while(loop <= n / 2){
            i = startX; j = startY;
            for(; j < n - offset; j++){ // 1. n - offset, not n
                nums[i][j] = count++;
            }
            for(; i < n - offset; i++){
                nums[i][j] = count++;
            }
            for(; j >= offset; j--){    // 2. offset, not 0
                nums[i][j] = count++;
            }
            for(; i >= offset; i--){
                nums[i][j] = count++;
            }
            startX++; startY++; offset++; loop++;
        }
        if(n % 2 == 1){     // n 为奇数时, 单独处理矩阵中心的值
            nums[startX][startY] = count;
        }
        return nums;
    }
}

2 - 链表（7题）

2.1 理论基础

链表的类型

  • 单链表

一种通过指针串联在一起的线性结构，每一个节点由两部分组成：

一个是数据域一个是指针域，最后一个节点的指针域指向 null。链表的入口节点称为链表的头结点，也就是 head。

  • 双链表

每一个节点有两个指针域，一个指向下一个节点，一个指向上一个节点。双链表既可以向前查询也可以向后查询。

  • 循环链表

顾名思义，就是链表首尾相连。循环链表可以用来解决约瑟夫环问题。

链表的存储方式

数组是在内存中是连续分布的，但是链表在内存中可不是连续分布的。链表是通过指针域的指针链接内存中的各个节点。

链表的定义

链表节点的定义，很多同学在面试的时候都写不好。

这是因为平时在刷 leetcode 的时候，链表的节点都默认定义好了，直接用就行了，所以没有注意到链表的节点是如何定义的。

链表的操作

  • 删除节点

删除 D 节点，只要将 C 节点的 next 指针 指向 E 节点就可以了。

D 节点依然存留在内存里，而 Java 有自己的内存回收机制，不用自己手动释放。

  • 添加节点

可以看出，链表的删除和添加都是 O(1) 操作，也不会影响到其他节点。

但要是删除第四个节点，需要从头节点查找到第三个节点通过 next 指针置空操作，查找的时间复杂度是 O(n)。

性能分析

数组的长度是固定的，如果想改动数组的长度，就需要重新定义一个新的数组。

链表的长度是不固定的，并且可以动态增删， 适合数据量不固定，频繁增删，较少查询的场景。

2.2 移除链表元素

  • 题目

力扣题目链接

题意：删除链表中等于给定值 val 的所有节点。

示例 1： 输入：head = [1,2,6,3,4,5,6], val = 6 输出：[1,2,3,4,5]

示例 2： 输入：head = [], val = 1 输出：[]

示例 3： 输入：head = [7,7,7,7], val = 7 输出：[]

  • 思路

这里以链表 1 4 2 4 来举例，移除元素 4。

但如果删除的是头结点，又该怎么办呢？

  1. 直接使用原来的链表来进行删除操作

    只要将头结点向后移动一位就可以，这样就从链表中移除了一个头结点。

  2. 设置一个虚拟头结点在进行删除操作

    这样原链表的所有节点就都可以按照统一的方式进行移除了。

  • 实现

xxxxxxxxxx
class Solution {
    public ListNode removeElements(ListNode head, int val) {
        if(head == null){
            return head;
        }
        ListNode dummy = new ListNode(-1, head); // 虚拟头节点
        ListNode a = dummy;
        ListNode b = head;


        while(b != null){
            if(b.val == val){
                a.next = b.next;
            } else{
                a = b;
            }
            b = b.next;
        }
        return dummy.next;
    }
}

2.3 设计链表

  • 题目

力扣题目链接

题意：实现 MyLinkedList 类：

  • MyLinkedList() 初始化 MyLinkedList 对象。

  • int get(int index) 获取链表中下标为 index 的节点的值。如果下标无效，则返回 -1 。

  • void addAtHead(int val) 将一个值为 val 的节点插入到链表中第一个元素之前。新节点会成为链表的第一个节点。

  • void addAtTail(int val) 将一个值为 val 的节点追加到链表中作为链表的最后一个元素。

  • void addAtIndex(int index, int val) 将一个值为 val 的节点插入到链表中下标为 index 的节点之前。

    • 如果 index 等于链表的长度，则该节点将附加到链表的末尾

    • 如果 index 大于链表的长度，则该节点将不会插入到链表中

    • 如果 index 小于0，则在头部插入节点

  • void deleteAtIndex(int index) 如果下标有效，则删除链表中下标为 index 的节点。

示例：

xxxxxxxxxx
// 链表变为 head, size = 0
MyLinkedList myLinkedList = new MyLinkedList(); 
myLinkedList.addAtHead(1);      // 链表变为 1
myLinkedList.addAtTail(3);      // 链表变为 1->3
myLinkedList.addAtIndex(1, 2);    // 链表变为 1->2->3
myLinkedList.get(1);              // 返回 2
myLinkedList.deleteAtIndex(1);    // 链表变为 1->3
myLinkedList.get(1);              // 返回 3

xxxxxxxxxx
class MyLinkedList {
    int size;
    ListNode head;                              // 1. ListNode, not MyLinkedList


    // A. 初始化 MyLinkedList 对象
    public MyLinkedList() {
        size = 0;                               // 2. 0, not 1
        head = new ListNode(0);
    }


    // B. 获取链表中下标为 index 的节点的值。如果下标无效，则返回 -1
    public int get(int index) {
        if (index < 0 || index >= size) {
            return -1;
        }
        ListNode node = head;       
        for (int i = 0; i <= index; i++) {      // head 无索引, 后移 index+1 次到目标节点
            node = node.next;
        }
        return node.val;
    }


    // C. 将一个值为 val 的节点插入到链表中第一个元素之前。新节点会成为链表的第一个节点
    public void addAtHead(int val) {
        addAtIndex(0, val);
    }


    // D. 将一个值为 val 的节点追加到链表中作为链表的最后一个元素
    public void addAtTail(int val) {
        addAtIndex(size, val);                  // a. 如果 index 等于链表的长度，则该节点将附加到链表的末尾
    }


    // E. 将一个值为 val 的节点插入到链表中下标为 index 的节点之前
    public void addAtIndex(int index, int val) {
        if (index > size) {                     // b. 如果 index 大于链表的长度，则该节点将不会插入到链表中
            return;
        }
        if (index < 0) {                        // c. 如果 index 小于0，则在头部插入节点
            index = 0;
        }
        ListNode node = head;
        for(int i = 0; i < index; i++){         // head 无索引, 后移 index 次到目标节点的上一个节点
            node = node.next;
        }
        ListNode x = new ListNode(val);
        x.next = node.next;
        node.next = x;
        size++;                                 // 3. don't forget
    }


    // F. 如果下标有效，则删除链表中下标为 index 的节点
    public void deleteAtIndex(int index) {
        if(index >= size || index < 0){         // 4. >=, not >
            return;
        }
        ListNode node = head;
        for(int i = 0; i < index; i++){
            node = node.next;
        }
        node.next = node.next.next;
        size--;                                 // 5. don't forget
    }
}

        ListNode N = dummy;

2.6 删除链表的倒数第 N 个节点

  • 题目

力扣题目链接

给你一个链表，删除链表的倒数第 n 个结点，并且返回链表的头结点。

示例 1：

xxxxxxxxxx
输入：head = [1,2,3,4,5], n = 2
输出：[1,2,3,5]

示例 2：

xxxxxxxxxx
输入：head = [1], n = 1
输出：[]

  • 思路

双指针的经典应用，让 fast 移动 n 步，然后让 fast 和 slow 同时移动，直到 fast 指向链表末尾。删掉 slow 所指向的节点就可以了。

分为如下两步：① 定义 fast 指针和 slow 指针，初始值为虚拟头结点（方便删除头结点）

② fast 首先走n + 1步，因为只有这样同时移动的时候 slow 才能指向删除节点的上一个节点（方便做删除操作）

  • 实现

xxxxxxxxxx
class Solution {
    public ListNode removeNthFromEnd(ListNode head, int n) {
        ListNode dummy = new ListNode(-1);  
        dummy.next = head;
        ListNode fast = dummy;              // 指向虚拟头节点
        ListNode slow = dummy;


        for (int i = 0; i <= n; i++) {      // 先走 n+1 步
            fast = fast.next;
        }
        while (fast != null) {
            fast = fast.next;
            slow = slow.next;
        }
        slow.next = slow.next.next;
        return dummy.next;                  // don't forget

public class Solution {
    public ListNode getIntersectionNode(ListNode headA, ListNode headB) {
        ListNode x = headA;
        ListNode y = headB;
        int lenA = 0, lenB = 0;
        while (x != null) {         // 求链表 A 的长度
            lenA++;
            x = x.next;
        }
        while (y != null) {         // 求链表 B 的长度
            lenB++;
            y = y.next;
        }
        x = headA;
        y = headB;
        if (lenB > lenA) {          
            int tmpLen = lenA;      // 保证 lenA > lenB
            lenA = lenB;
            lenB = tmpLen;
            ListNode tmpNode = x;   // 保证 x(y) 指向 headA(B)
            x = y;
            y = tmpNode;
        }


        int gap = lenA - lenB;
        while (gap-- > 0) {         // gap-- > 0, not gap--         
            x = x.next;
        }
        while (x != null) {

                return x;
            } else{
                x = x.next;
                y = y.next;
            }
        }

① 判断链表是否环

② 如果有环，如何找到这个环的入口

x = n (y + z) - y


# 1. 当 n = 1 时, x = z
# 即: 从头结点、相遇节点各出发一个指针, 相遇的时候就是环形入口的节点
# 2. 当 n > 1 时, 效果相同, 只不过是多绕了 n-1 圈

public class Solution {
    public ListNode detectCycle(ListNode head) {
        ListNode slow = head;
        ListNode fast = head;
        // 1. not fast != null && slow != null
        // 2. not fast.next != null && fast != null
        while (fast != null && fast.next != null) {     
            slow = slow.next;
            fast = fast.next.next;
            if (slow == fast) { // 有环
                ListNode index1 = head;
                ListNode index2 = fast;
                while (index1 != index2) {
                    index1 = index1.next;
                    index2 = index2.next;
                }
                return index1;
            }
        }
        return null; // 无环, 或空链表
    }
}

3.1 理论基础

哈希表

哈希表是根据关键码的值而直接进行访问的数据结构。关键码就是数组的索引下标，然后通过下标直接访问数组中的元素。

那么哈希表能解决什么问题呢？一般哈希表都是用来快速判断一个元素是否出现集合里。

例如：查询一个名字是否在这所学校里。要枚举的话时间复杂度是 O(n)，但如果使用哈希表的话， 只需要 O(1) 就可以做到。

  • 哈希函数

哈希函数，把学生的姓名直接映射为哈希表上的索引，然后就可以通过查询索引下标快速知道这位同学是否在这所学校里了。

一般 hashcode 是通过特定编码方式，可以将其他数据格式转化为不同的数值，这样就把学生名字映射为哈希表上的索引数字了。

哈希碰撞

如果小李和小王都映射到了索引下标 1 的位置，这一现象叫做哈希碰撞。一般有两种解决方法， 拉链法和线性探测法。

  • 拉链法

拉链法，将发生冲突的元素都被存储在链表中。，这样我们就可以通过索引找到小李和小王了。

拉链法要选择适当的哈希表大小，这样既不会因为数组空值而浪费大量内存，也不会因为链表太长而在查找上浪费太多时间。

  • 线性探测法

线性探测法， 依靠哈希表中的空位来解决碰撞问题。例如冲突的位置放了小李，那么就向下找一个空位放置小王的信息。

所以要求 tableSize 一定要大于 dataSize ，要不然哈希表上就没有空置的位置来存放冲突的数据了。

常见的三种哈希结构

当我们想使用哈希法来解决问题的时候，我们一般会选择如下三种数据结构：数组、set （集合）、map（映射）。

在 C++ 中，set 和 map 分别提供以下三种数据结构，其底层实现以及优劣如下表所示：

红黑树是一种平衡二叉搜索树，所以 key 值是有序的，但不可以修改，改动 key 值会导致整棵树的错乱，所以只能删除和增加。

① set：当我们要使用 set 来解决哈希问题的时候，优先使用 unordered_set，因为它的查询和增删效率是最优的；

如果需要集合是有序的，那么就用 set；如果要求不仅有序还要有重复数据的话，那么就用 multiset。

② map：一个 key value 的数据结构，map 中对 key 有限制，对 value 没有限制，因为 key 的存储使用红黑树实现。

其他语言例如：java 里的 HashMap ，TreeMap 都是一样的原理。可以灵活贯通。

③ 虽然 set、 multiset 的底层实现是红黑树不是哈希表，不过使用方式还是哈希法的使用方式，即 key 和 value。

所以使用这些数据结构来解决映射问题的方法，我们依然称之为哈希法。map 也是一样的道理。

  • 总结

总结一下，当我们遇到了要快速判断一个元素是否出现集合里的时候，就要考虑哈希法。

但是哈希法也是牺牲了空间换取了时间，因为我们要使用额外的数组，set 或者是 map 来存放数据，才能实现快速的查找。

  • 题目

力扣题目链接

给你一个整数数组 nums ，返回所有三元组 [nums[i], nums[j], nums[k]] ：

输出：[[-1,-1,2],[-1,0,1]]

  • 思路

其实这道题目使用哈希法并不十分合适，使用双指针法要比哈希法高效一些，那么来讲解一下具体实现的思路。

首先将数组排序，然后有一层 for 循环，i 从下标 0 的地方开始，同时定义 left 在 i+1 的位置上，定义 right 在数组结尾的位置上。

如果 nums[i] + nums[left] + nums[right] > 0 说明 此时三数之和大了，所以 right 就应该向左移动，这样才能让三数之和小一些。

如果 nums[i] + nums[left] + nums[right] < 0 说明 此时三数之和小了，所以 left 就应该向右移动，这样才能让三数之和大一些。

  • 实现

class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();
        Arrays.sort(nums);
        for (int i = 0; i < nums.length - 2; i++) {
            int a = nums[i];
            if (a > 0) break;
            if (i > 0 && a == nums[i - 1]) 
                continue;               // a 去重
            int left = i + 1;
            int right = nums.length - 1;
            while (right > left) {
                int b = nums[left];
                int c = nums[right];
                if (a + b + c > 0) {
                    right--;
                } else if (a + b + c < 0) {
                    left++;

                    while (right > left && b == nums[left + 1])

                    while (right > left && c == nums[right - 1])

    public List<List<Integer>> fourSum(int[] nums, int target) {
        List<List<Integer>> result = new ArrayList<>();
        Arrays.sort(nums);
        for (int i = 0; i < nums.length; i++) {
            int a = nums[i];
            if (a > target && a >= 0) break;
            if (i > 0 && a == nums[i - 1]) {
                continue;                   // 1. 去重 a
            }
            for (int j = i + 1; j < nums.length; j++) {
                int b = nums[j];
                if (a + b > target && a + b >= 0) break;
                if (j > i + 1 && b == nums[j - 1]) {
                    continue;               // 2. 去重 b
                }
                int left = j + 1;
                int right = nums.length - 1;
                while (right > left) {
                    int c = nums[left];
                    int d = nums[right];

                        right--;
                    } else if (a + b + c + d < target) {
                        left++;
                    } else {
                        result.add(Arrays.asList(a, b, c, d));

                            left++;         // 3. 去重 c
                        while (right > left && d == nums[right - 1]) 
                            right--;        // 4. 去重 d
                        right--;

                    }
                }
            }
        }

3.10 总结

数组作为哈希表

数组就是简单的哈希表，但是数组的大小是受限的。一些同学可能想，用数组干啥，都用 map 不就完事了。

但使用 map 的空间消耗要比数组大一些，因为要维护红黑树或者符号表，而且还要做哈希函数的运算。

所以数组更加简单直接有效，一些应用场景就是为数组量身定做的。

set 作为哈希表

当题目没有限制数值大小的时候，就无法使用数组来做哈希表了。主要因为如下两点：

  • 数组的大小是有限的，受到系统栈空间（不是数据结构的栈）的限制。

  • 如果数组空间够大，但哈希值比较少、特别分散、跨度非常大，使用数组就造成空间的极大浪费。

如果题目需要对数据进行排序的话，就选择 set 和 multiset，底层实现都是红黑树，且 key 是有序的。

如果题目不需要对数据进行排序的话，就选择 unordered_set，底层实现是哈希表，且读写效率最高。

map 作为哈希表

来说一说使用数组和 set 来做哈希法的局限。

  • 数组的大小是受限制的，而且如果元素很少，而哈希值太大会造成内存空间的浪费。

  • set 是一个集合，里面放的元素只能是一个 key，而如果还需要保存对应的 value，就不能用 set。

虽然 map 是万能的，但是像三数之和、四数之和这样的题目非常麻烦，需要去重导致代码效率很低，所以推荐双指针的解法。

class Solution {
    public String reverseStr(String s, int k) {
        char[] array = s.toCharArray();
        for (int i = 0; i < array.length; i += (2 * k)) {
            if (i + k <= array.length) {                // 1. >= k 个, 反转前 k 个字符
                reverse(array, i, i + k - 1);
            } else {    
                reverse(array, i, array.length - 1);    // 2. < K 个, 反转全部字符

    }

4.6 实现 strStr()

  • 题目

力扣题目链接

给你两个字符串 haystack 和 needle ，请你在 haystack 字符串中找出 needle 字符串的第一个匹配项的下标。

示例：

xxxxxxxxxx
输入：haystack = "sadbutsad", needle = "sad"
输出：0

  • 思路

KMP 的经典思想就是：当出现字符串不匹配时，可以记录一部分之前已经匹配的文本内容，利用这些信息避免从头再去做匹配。

前缀表是用来回退的，它记录了模式串与主串不匹配的时候，模式串应该从哪里开始重新匹配。

next 数组就可以是前缀表，但是很多实现都是把前缀表统一减一之后作为 next 数组。

public class Solution {     
    public int strStr(String haystack, String needle) {
        int[] next = new int[needle.length()];
        getNext(next, needle);
        int j = -1;
        // 看 GIF 理解, i 是上面的指针, j+1 是下面的指针
        for (int i = 0; i < haystack.length(); i++) {
            while (j >= 0 && haystack.charAt(i) != needle.charAt(j + 1)) 
                j = next[j];        
            if (haystack.charAt(i) == needle.charAt(j + 1)) 
                j++;                
            if (j == needle.length() - 1) 
                return i - needle.length() + 1;
        }
        return -1;
    }   // 直接复制到下面, 小改一下
    private void getNext(int[] next, String s) {
        int j = -1;
        next[0] = j;
        for (int i = 1; i < s.length(); i++) {
            while (j >= 0 && s.charAt(i) != s.charAt(j + 1)) 
                j = next[j];
            if (s.charAt(i) == s.charAt(j + 1)) 
                j++;
            next[i] = j;        // don't forget
        }
    }
}

6 - 栈与队列（7题）

6.1 理论基础

C++

  • 栈

先进后出的数据结构，不允许有遍历行为，不提供迭代器，SGI STL 中以 deque 为缺省情况下的底部结构。

可以指定 vector 为底层实现，初始化语句如下：stack<int, vector<int>> third;

所以 STL 栈不被归类为容器，而被归类为 container adapter（ 容器适配器）。

  • 队列

先进先出的数据结构，不允许有遍历行为，不提供迭代器，SGI STL 中以 deque 为缺省情况下的底部结构。

可以指定 list 为底层实现，初始化 queue 的语句如下：queue<int, list<int>> third;

所以 STL 队列不被归类为容器，而被归类为 container adapter（ 容器适配器）。

Java

  • 栈

虽然 Stack 类继承自 Vector 类，而 Vector 类使用数组。实际上，更常见的是使用数组或链表实现栈。

① 数组实现：入栈操作将元素添加到数组的末尾，出栈操作则移除数组末尾的元素。

② 链表实现：入栈操作将元素插入到链表的头部，出栈操作则移除链表头部的元素。

  • 队列

队列的底层实现通常使用数组、链表或 deque。提供了 ArrayDeque 和 LinkedList 两种常见的实现类。

① 数组实现：入队操作将元素添加到数组的末尾，出队操作则移除数组的头部元素。

② 链表实现：入队操作将元素添加到链表的末尾，出队操作则移除链表的头部元素。

            else if (c == '{') stack.push('}');
            else if (c == '[') stack.push(']');
            else if (stack.isEmpty() || c != stack.pop()) return false;
        }
        return stack.isEmpty();     // 2. stack.isEmpty(), not !stack.isEmpty()
    }
}

给你一个整数数组 nums，有一个大小为 k 的滑动窗口从数组的最左侧移动到数组的最右侧。

你只可以看到在滑动窗口内的 k 个数字。滑动窗口每次只向右移动一位。返回 滑动窗口中的最大值。

示例 ：

输入：nums = [1,3,-1,-3,5,3,6,7], k = 3
输出：[3,3,5,5,6,7]
解释：
滑动窗口的位置                  最大值
---------------               -----
[1  3  -1] -3  5  3  6  7       3
1 [3  -1  -3] 5  3  6  7       3
1  3 [-1  -3  5] 3  6  7       5
1  3  -1 [-3  5  3] 6  7       5
1  3  -1  -3 [5  3  6] 7       6
1  3  -1  -3  5 [3  6  7]      7

  • 思路

  • 实现

xxxxxxxxxx
class Solution {
    private class Myque {
        Deque<Integer> dq;
        
        public Myque() {                                    // 1. not class
            dq = new ArrayDeque<>();
        }
        void pop(int x) {
            if (!dq.isEmpty() && x == dq.peekFirst()) {     // 2. not peekFront
                dq.pollFirst();
            }
        }
        void push(int x) {
            while (!dq.isEmpty() && x > dq.peekLast()) {
                dq.pollLast();
            }
            dq.offerLast(x);
        }
        int front() {
            return dq.peekFirst();
        }
    }
    
    public int[] maxSlidingWindow(int[] nums, int k) {
        Myque q = new Myque();
        int n = nums.length, j = 0;                         // 3. not length()
        int[] res = new int[n - k + 1];


        for (int i = 0; i < k; i++) {                       // 初始化窗口
            q.push(nums[i]);
        }
        res[j++] = q.front();
        for (int i = k; i < n; i++) {                       // 滑动窗口过程
            q.pop(nums[i - k]);
            q.push(nums[i]);
            res[j++] = q.front();
        }
        return res;
    }
}

6.8 前 K 个高频元素

  • 题目

力扣题目链接

给你一个整数数组 nums 和一个整数 k ，请你返回其中出现频率前 k 高的元素。你可以按 任意顺序 返回答案。

示例 :

xxxxxxxxxx
输入: nums = [1,1,1,2,2,3], k = 2
输出: [1,2]

  • 思路

首先统计元素出现的频率，可以使用 map 来进行统计。然后是对频率进行排序，优先级队列就是一个披着队列外衣的堆。

我们要用小顶堆，因为要统计最大前 k 个元素，只有小顶堆每次将最小的元素弹出，最后小顶堆里积累的才是前 k 个最大元素。

  • 实现

xxxxxxxxxx
class Solution {
    public int[] topKFrequent(int[] nums, int k) {
        // 1. 统计每个元素的频率
        Map<Integer, Integer> map = new HashMap<>();
        for (int n : nums)
            map.put(n, map.getOrDefault(n, 0) + 1);     // error: map.add()
        // 2. 保存出现频率前 k 高的元素                     // error: PriorityHeap, a.getValue()
        PriorityQueue<Integer> minHeap = new PriorityQueue<>((a, b) -> map.get(a) - map.get(b));
        for (int key : map.keySet()) {                  // error: map.setKey()
            minHeap.offer(key);
            if (minHeap.size() > k) 
                minHeap.poll();
        }
        // 3. 从堆中提取最上面的 k 个元素
        int[] res = new int[k];
        int i = 0;
        while (!minHeap.isEmpty())
            res[i++] = minHeap.poll();
        return res;
    }
}

6.9 总结

理论基础

栈与队列的底层实现，很多同学都比较模糊。面试题：栈里面的元素在内存中是连续分布的么？

  • 栈是容器适配器，底层容器使用不同的容器。

  • 缺省情况下，默认底层容器是 deque，在内存中的数据分布是不连续的。

栈在系统中的应用：举个例子，cd 进入目录的命令：cd a/b/c/../../ 最后进入 a 目录。

这就是栈的应用：每一次递归调用都会把函数的局部变量、参数值和返回地址等压入调用栈中。

栈的经典问题

例：6.4 括号匹配问题、6.5 字符串去重问题、6.6 逆波兰表达式问题

队列的经典问题

例：6.7 滑动窗口最大值问题、6.8 求前 K 个高频元素

① 滑动窗口：队列没有必要维护窗口里的所有元素，只需要维护有可能成为窗口里最大值的元素就可以了。

单调队列不是一成不变的，而是不同场景不同写法，总之要保证队列里单调递减或递增的原则。

② 优先级队列：就是一个披着队列外衣的堆。最小堆是一棵完全二叉树，树中每个结点的值都不大于其左右孩子的值。
